## IMPORTANT ##
After extruding the single vertex and converting to curve, 
once you convert to mesh you CANNOT convert back. If you 
decide to convert back you can no longer edit depth or resolution.

Anything marked with a ? is unknown

Because we are working with single layer objects, size no longer matters
or effects how it looks when resized in game.

For unknown variables, please notate the when an object is first created 
and once correct abide by that standard.

When using the ruler tool, use CTRL + Left Click to snap to an objects surface.
Practice first so you understand how it works.

UPDATE:
	New specification added for scenes in redot pertaining to how traps, lights, etc should be titled and function

## Measurements ##

Measurements (Not tracked by blender, use ruler tool):
	NOTE: Add new entries as needed, add custom fields as needed

	Symbols: 
			  | No symbol; Cannot change
			~ | inconsistent number or relies on things like pipe length or circle cut placement
			* | Any number a multiple of the supplied
			> | Scales up to a desired size
			< | Scales down to a desired size
			

	STANDADRS:
		Total Length (per node placement): * 4 Meters
		Pipe Resolution: 6
		Inner Diameter: * 0.75 Depth OR 1.5 Inner Diameter (equal when measured)
		Angles: 1 Meter Offset @90° scaled to angle (45° == 0.5m Offset)
		Circle Cut: See Flare Depth
		Flare Width: * 0.15 Meters
		Flare Depth: *~ 0.26 Meters
		Transitions: Flare Width >< New Flare Width
		Shear Offset: 1 Meter

## NOTES ##
	Pipes should be long enough to accomodate for player speed
	
	Flared ends should be measured and notated to a standard size
	
	To find the length of your flared end, before finalizing the length of extrusion will be noted at 
		the top of the screen as such:
			D: 0.0000 m (0.000 0m normale)
		M as in meters
	
	Pipe lengths should be measured and notated to a standard length to keep total
		length consistent between objects. Use CTRL + Left Click with the ruler tool.
		
		In edit/object mode the measure tool is avaliable as a square with an angle in the 
		corner on the left side of the screen

## Scene Standards ##

# - A numbered object


Declarations:
	Lights will always be underneath a Node3D object named "Lighting"
	Traps will always exist under a Node3D named "Trap"
	Animation Players retain their default name
	Trap Scripts need to be attached to the root node

Root Scene Layout:
	ROOT (Node3D):
		player (child scene)
		Goal (child scene)
		Level (Node3D):
			Dungeon (child scene, any full level object)

Piece Scene Layouts:
	Root (Node3D, named Dungeon):
		AnimationPlayer
		SpawnPoint - where the player spawns
		GoalPoint - where the goal spawns
		Structure (MeshInstance3D, name):
			StaticBody3D:
				CollisionShape3D (Trimesh)
			Structures Inside Structure (water, ledges, rooms, decorations, etc)
		Lighting (Node3D)
		Trap# (Child Scene Instance)

Trap Scene Layouts:
	Root (Node3D, named Trap):
		Node3D (name of trap):
			DetectionArea# (Area3D):
				DetectionZone (CollisionShape3D, single convex)
			DamageArea# (Area3D):
				DamageZone (CollisionShape3D, single convex):
			Node3D (named Construction):
				Node3D (name of object):
					MeshInstance3D# - NOTE: Does not need collision

Level Scene Layouts:
	Root (Node3D, name of level)
		World Enviorment:
			DirectionalLight3D (if outside)
		Player (Child Scene Instance)
		Piece (Name and/or Number, Child Scene Instance)
