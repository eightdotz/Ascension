## IMPORTANT ##
After extruding the single vertex and converting to curve, 
once you convert to mesh you CANNOT convert back. If you 
decide to convert back you can no longer edit depth or resolution.

Anything marked with a ? is unknown

Because we are working with single layer objects, size no longer matters
or effects how it looks when resized in game.

For unknown variables, please notate the when an object is first created 
and once correct abide by that standard.

When using the ruler tool, use CTRL + Left Click to snap to an objects surface.
Practice first so you understand how it works.

	New specification added for scenes in redot pertaining to how traps, lights, etc should be titled and function

## Measurements ##

Measurements (Not tracked by blender, use ruler tool):
	NOTE: Add new entries as needed, add custom fields as needed

	Symbols: 
			  | No symbol; Cannot change
			~ | inconsistent number or relies on things like pipe length or circle cut placement
			* | Any number a multiple of the supplied
			> | Scales up to a desired size
			< | Scales down to a desired size
			

	STANDADRS:
		Total Length (per node placement): * 4 Meters
		Pipe Resolution: 6
		Inner Diameter: * 0.75 Depth OR 1.5 Meters Inner Diameter (equal when measured)
		Room Entrance and Exit Diameter: 0.6 Meters (For existing rooms) and 1.5 Meters for new rooms	
		Angles: 1 Meter Offset @90° scaled to angle (45° == 0.5m Offset)
		Circle Cut: See Flare Depth
		Flare Width: * 0.15 Meters
		Flare Depth: *~ 0.26 Meters
		Transitions: Flare Width >< New Flare Width
		Shear Offset: 1 Meter

## NOTES ##
	Pipes should be long enough to accomodate for player speed
	
	Flared ends should be measured and notated to a standard size
	
	To find the length of your flared end, before finalizing the length of extrusion will be noted at 
		the top of the screen as such:
			D: 0.0000 m (0.000 0m normale)
		M as in meters
	
	Pipe lengths should be measured and notated to a standard length to keep total
		length consistent between objects. Use CTRL + Left Click with the ruler tool.
		
		In edit/object mode the measure tool is avaliable as a square with an angle in the 
		corner on the left side of the screen

## Scene Standards ##

Overlap Detection is set to collision layer and mask 5 on ready

Scene 'root.tscn' (under '/scenes'):
	Description:
		root.tscn or ROOT is the level manager. This script is resposible for loading and deloading levels along with teleporting
		the player and goal to their proper position. This script contains four nodes with player and Level being persistant across 
		the gameplay

	ROOT (Node3D):
		Emits 'level_changed' (See Signals). 
		Attached script 'root.gd' (See Scripts) which is the level managing script.
		The level manager, instantiates levels to be generated under the Level node

	|-  Level (Node3D):
			The place ROOT stores and frees instantiated levels.

	|-  Goal (Child Scene 'goal.tscn', Node3D):
			Emits 'level_completed' (See Signals).
			Attached script 'goal.gd' (See Scripts).
			s the only instance of 'Goal' anywhere in the project. Goal is teleported to the last piece of a level on load.

	|-  Player (Child Scene 'player.tscn', CharacterBody3D):
			Emits 'on_click' (See Signals).
			Attached 'player.gd'. Provides customization options for the player and game apperence (under Shaders)
			Is the only instance of 'Player' anywhere in the project. The player is teleported to the first piece of a level on load.

Scene 'Level.tscn' (under '/scenes/main'):
	Description:
		Level.tscn is responsible for generating levels by looping through and instantiating level pieces. 
		Contains the spawn points for goals and players and requires a 'type' and 'biome' to be set. These are required to 
		influence player behavior and which folders to search through. Levels also contain the world enviorment which 
		has modifiers for ambient light and fog to set the tone of a level
	
	Dungeon (Node3d):
		Attached script 'level.gd'. Controls the generation of levels by instantiating pieces under the Pieces Node.
	
	|-   WorldEnviorment:
			Defines the look of the level with fog, ambient light, camera compositing, fog, reflections, etc.
			
	
	|-  SpawnPoint (Child Scene 'SpawnPoint.tscn', Node3D):
			Placed where the player will spawn at the level, usually at the starting joint of the first piece.
			It works by moving this node to the global coords of the first joint of the first piece where the player will be teleported.
	
	|-  GoalPoint (Child Scene 'SpawnPoint.tscn', Node3D):
			Placed where the goal will be placed, usually at the final joint of the last piece
			It works by moving this node to the global coords of the first joint of the first piece.
			
	|-  Pieces (Node3D):
			Contains all the pieces being instantiated. All pieces should be freed when the level itself is freed by the level manager.
 
Piece Scenes (under '/scenes/biomes'):
	Description:
		Piece scenes are any scene that exsits to construct levels. These scenes make up big and small structures
		Piece scenes require Start and End nodes made of 'Joint' scenes and a general structure. All else is optional.
		It is encouraged that these scenes are filled with parts that funcionally make the gameplay intersting. This 
		could be surfaces for players to navigate or traps for players to avoid.
		
	NamedNode (Node3D of any name):
		Attached script 'level_piece.gd'.
		This node can randomize the spawning of traps.

	|-  MainBody (MeshInstance3D):
			Primary structure of a piece. Think of a room but not the things inside it.
			Decorations, traps, platforms, etc. Will be placed inside of this node.
			
			|-  StaticBody3D
			
					|- CollisionShape3D
			
	|-  Lighting (Node3D):
			Contains all lighting of this scene. This can be used to create atmosphere by stratigically hiding this node or its children
			as well as editing their modifiers dynamically

	|-  Start (Child Scene 'joint.tscn', Node3D):
			The enterance to the structure. Where a player spawn point can be placed. This node has a face on it to indicate the front
		
	|-  End (Child Scene 'joint.tscn', Node3D):
			The exit to the structure. Where a goal spawn point can be placed. This node has a face on it to indicate the front.
	
	|-  Checkpoint
			Area3D that tracks what pipe the current player has just entered
	
	|-  OverlapChecks (Node3D):
			Organizer for overlap detectors
			|-  OverlapDetector#
					Detects whether two Area3Ds overlap on layer 5

Trap Scenes (under '/traps'):
	Description:
		Traps are used to either A: Kill the player; B: Harm the player; C: Steal the player; D: Inconvience the player. Traps can be 
		anything that thematically fit the level however they should not be outright unfair or hidden in a way that doesnt 
		communicate the danger they present. There should always be a way out. Traps are always instantiated into level pieces.
	
	Trap (Node3D):
		Contains the primary trap script for controlling how a trap activates, behaves, and effects its surroundings.
		The general script for these will be marked with a '!' (like '!GenericTrap.gd') defining a catagory they posses
		however if required a new script can be made for unique traps that arent able to be combined with others in terms 
		of behavior
		
	|-  DetectionArea (Area3D):
			Container and signal provider for detecting collision for triggers
			
			|-  DetectionZone (CollisionShape3D)
			
	|-  DamageArea (Area3D):
			Container and signal provider for detecting collision for effecting the player
			
			|-  DamageZone (CollisionShape3D)
			
	|-  Construction (Node3D):
			The general design of the trap. This can be meshes with textures, glow, particles, etc.
			Any design element fall under this node.
